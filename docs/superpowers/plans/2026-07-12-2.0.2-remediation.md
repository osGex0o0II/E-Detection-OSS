# E-Detection 2.0.2 Remediation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix all confirmed review findings, add executable regressions, align every version surface to 2.0.2, and publish an explicitly unsigned `v2.0.2` release with verified assets.

**Architecture:** Keep the existing WinUI shell and native core, but extract small reusable execution, semantic-version, and path-safety units where behavior must be tested directly. Extend the headless console project into a named regression harness and add PowerShell behavioral smokes for script/settings boundaries.

**Tech Stack:** C# 14, .NET 10, WinUI 3, PowerShell 7, GitHub Actions, Inno Setup 6.

## Global Constraints

- The release is intentionally unsigned and every trust-related message must say so.
- SHA-256 is an integrity check, not publisher authentication.
- No Python runtime or bridge may return to the package.
- Every production behavior change requires a focused failing regression first.
- Destructive path tests operate only in generated directories under `artifacts/desktop/script-safety-smoke`.
- The final repository version is `2.0.2`; assembly/file versions are `2.0.2.0`.

```csharp
internal static class RegressionSuite
{
    private static readonly Dictionary<string, Func<Task>> Tests = new(StringComparer.OrdinalIgnoreCase);

    public static async Task RunAsync(string name)
    {
        var selected = name.Equals("all", StringComparison.OrdinalIgnoreCase)
            ? Tests.OrderBy(static pair => pair.Key).ToList()
            : Tests.TryGetValue(name, out var test)
                ? [new KeyValuePair<string, Func<Task>>(name, test)]
                : throw new ArgumentException($"Unknown regression: {name}");
        foreach (var (testName, test) in selected)
        {
            await test();
            Console.WriteLine($"PASS {testName}");
        }
    }
}
```

- [ ] **Step 2: Add fixture and assertion helpers**

```csharp
internal static class TestSupport
{
    public static void Equal<T>(T expected, T actual, string message)

            throw new InvalidOperationException($"{message} Expected={expected} Actual={actual}");
    }

    public static string CreateFixtureDirectory(string name)
    {
        var path = Path.Combine(Path.GetTempPath(), $"EDetection-{name}-{Guid.NewGuid():N}");
        Directory.CreateDirectory(path);
        return path;
    }
}
```
